<objective>
Replace the existing `Home.tsx` placeholder ("TPC Dashboard — being
built next") with the CRM v0.5 triage inbox. Read from `crm_triage_queue`
view, sort by age-bumped effective priority, surface Refresh button +
toast UX.

Files to create:

- `src/hooks/useCrmTriage.ts` — TanStack Query hook
  - Reads `crm_triage_queue` view via Supabase client
  - Applies age-bump rules **client-side**:
    - STANDARD + age_days ≥ 5 → effective_priority = HIGH
    - LOW + age_days ≥ 7 → effective_priority = STANDARD
    - LOW + age_days ≥ 14 → effective_priority = HIGH
  - Sort: effective_priority (HIGH→STD→LOW), then received_at ASC within bucket
  - Returns `{ threads: TriageRow[], isLoading, refetch }`
- `src/hooks/useCrmTriage.test.tsx` — unit tests covering all four
  age-bump rules + sort behaviour
- `src/components/crm/CRMInbox.tsx` — main inbox component
  - Renders `useCrmTriage()` rows in a table
  - Refresh button top-right: triggers POST to `/api/crm-poll` then
    `refetch()`. Spinner during. Toast on success/failure.
  - Empty state: illustration + "Inbox zero — no open consignments to
    triage" + Refresh button
  - Row expand: full body + classifier rationale
- `src/components/crm/PriorityChip.tsx` — red/yellow/green pill
  - HIGH = red, STANDARD = yellow, LOW = green
  - Uses dashboard's existing chip pattern if one exists; else
    Tailwind classes
- `src/components/crm/DeptTags.tsx` — gray inline pills
  - Renders department[] as small chips
  - D-036 5-tag set
- `src/components/crm/EmptyState.tsx` — empty illustration + copy
- `src/pages/Home.tsx` — **replace entire file** with `<CRMInbox />`
  render. Existing placeholder content removed; archive note in commit
  message
- `src/components/crm/__tests__/CRMInbox.test.tsx` — integration test
  with mocked hook returning sample data; verify priority sort order,
  age-bump display, row expand, Refresh button click triggers POST
- `src/components/crm/__tests__/PriorityChip.test.tsx` — color mapping
- `src/components/crm/__tests__/DeptTags.test.tsx` — multi-tag rendering
