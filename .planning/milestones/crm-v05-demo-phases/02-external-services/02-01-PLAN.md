<objective>
Build the three external-service clients the poller depends on: Streak
(box list + stage), Gmail (body fetch via OAuth refresh-token), and the
LLM classifier (Gemini 2.5 Flash direct, not via tpc-ai-proxy — proxy
requires Supabase JWT which the Vercel Function won't have).

Files to create:

- `src/services/crm/streakApi.ts`
  - `listOpenBoxes(pipelineKey, closedStageKeys[])` → `StreakBox[]`
  - Sequential fetch + fail-fast on 429/5xx
  - Auth: `Authorization: Bearer ${STREAK_API_KEY}` header
  - Closed-stage heuristic: filter by name pattern
    `/^(closed|won|lost|archived|done|completed)/i` if
    `STREAK_CLOSED_STAGE_KEYS` env not set
- `src/services/crm/gmailApi.ts`
  - OAuth refresh-token client using `googleapis` SDK
  - `getThreadBody(threadId)` → `string` (concatenated message body text)
  - **Verb allowlist enforced at module top:** only `messages.list` +
    `messages.get` allowed. Throw `GmailVerbForbidden` for anything else.
  - Reads `GMAIL_OAUTH_CLIENT_ID`, `GMAIL_OAUTH_CLIENT_SECRET`,
    `GMAIL_REFRESH_TOKEN`, `GMAIL_USER_EMAIL` from env
- `src/services/crm/crmClassifier.ts`
  - Direct Gemini 2.5 Flash call via `@google/generative-ai` SDK
  - Reads `GEMINI_API_KEY` from env
  - Input: `{subject, body, received_at, from_email, from_name}`
  - Output schema (JSON mode): `{department: string[], priority:
    'high'|'standard'|'low', rationale: string, model: string}`
  - Prompt embeds D-036 5-tag dept list, calibration target (~10% HIGH
    / 60% STD / 30% LOW), and the deterministic-override notes (VIP
    sender, deadline ≤7 days, empty-body LOW)
  - Soft cap: throw `ClassifierBudgetExceeded` if invoked > 200 times
    in same Vercel Function execution (per D-042 budget)
- `src/services/crm/types.ts` — shared types (`StreakBox`,
  `ClassificationResult`, `Department`, `Priority`)
- `src/services/crm/__tests__/` — vitest coverage:
  - `streakApi.test.ts` — happy path + 429 fail-fast + closed-stage
    filter behaviour
  - `gmailApi.test.ts` — happy path + verb-allowlist enforcement test
    (must throw when caller attempts `messages.modify` etc.)
  - `crmClassifier.test.ts` — output schema validation + override
    behaviour (mock LLM response, verify deterministic overrides
    applied correctly)

Add deps: `googleapis ^144`, `@google/generative-ai ^0.21`.

Success criteria:

- All three service files compile + lint clean.
- Vitest passes; verb-allowlist negative test asserts the throw.
- No real API calls in tests (all mocked).
- Type-check green: `npm run type-check`.
</objective>
