<objective>
Wire the three external services (from phase 02) into a single Vercel
Serverless Function that polls Streak → fetches Gmail body → classifies
via Gemini → writes to Supabase. Manual trigger only (the frontend
Refresh button POSTs to this endpoint).

Files to create:

- `api/crm-poll.ts` — Vercel Serverless Function (Node runtime, not Edge)
  - `export const maxDuration = 60` (Hobby plan; bump to 800 on Pro)
  - Method: `POST`. Returns 405 on other methods.
  - Auth: validate Supabase JWT from `Authorization: Bearer` header;
    look up profile via service-role client; reject if not admin /
    not is_active per D-040
  - Orchestration:
    1. `streakApi.listOpenBoxes(STREAK_PIPELINE_KEY, closedStageKeys)`
    2. For each box (sequential, no parallel — keeps Streak rate-friendly):
       a. Upsert `crm_threads` row (key on streak_box_key).
       b. Fetch full body via `gmailApi.getThreadBody(gmail_thread_id)`.
       c. Hash body content; compare to most-recent classification's
          source hash (stored in metadata). Skip classifier if unchanged.
       d. Call `crmClassifier.classify(...)`. Apply deterministic
          overrides (VIP allowlist via `STREAK_VIP_DOMAINS` env, deadline,
          empty-body).
       e. Mark previous classification `is_current = false`; insert new
          row with `is_current = true`.
       f. Log call to `private.api_usage` (tokens, cost, model,
          app_source = 'tpc-dashboard-crm-poll').
    3. Return `{ polled: N, classified: M, skipped_unchanged: K,
       errors: [] }`.
  - Soft budget: stop classifier calls after 200; remaining boxes
    counted as `deferred` in response.
- `vercel.json` — only if needed (likely unnecessary; Vercel auto-detects
  `/api/*.ts`). Include `maxDuration` if function-level export isn't
  picked up.
- `api/__tests__/crm-poll.test.ts` — integration-style test with all
  three services mocked. Verify:
  - Non-admin caller rejected (401/403)
  - Empty Streak response handled cleanly (no LLM calls)
  - Hash-skip behaviour (second poll on unchanged thread skips classifier)
  - Budget cap stops at 200 + reports `deferred`
  - `private.api_usage` row written on each classifier call
- Env requirements documented in `api/README.md` (or update root README):
  - `STREAK_API_KEY`, `STREAK_PIPELINE_KEY`, optional
    `STREAK_CLOSED_STAGE_KEYS` (csv), optional `STREAK_VIP_DOMAINS` (csv)
  - `GMAIL_OAUTH_CLIENT_ID`, `GMAIL_OAUTH_CLIENT_SECRET`,
    `GMAIL_REFRESH_TOKEN`, `GMAIL_USER_EMAIL`
  - `GEMINI_API_KEY`
  - `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY` (server-side; already
    in Vercel env from dashboard's existing scraper work)

Success criteria:

- `npm run build` succeeds with `api/` included.
- Vitest passes all crm-poll tests.
- Type-check green.
- Deployed to a preview URL — function returns clean
  `{ polled: 0, classified: 0, … }` with empty STREAK_API_KEY (env
  validation gates the actual fetch).
</objective>
