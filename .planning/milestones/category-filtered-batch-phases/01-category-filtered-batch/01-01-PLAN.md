<objective>
Implement the dashboard half of the cross-app "Category-Filtered Batch AI Generation" feature: a new top-level **Skip Reasons** chart panel on the `/extension` page that surfaces the 5 skip-reason breakdown columns added by the cataloger's schema migration.

Deliverables in this phase:

1. **Regenerate types** ŌĆö After cataloger phase pushes migration 007, run `npm run db:types` in this repo to pick up the 5 new `analytics_events` columns (`skipped_no_photos`, `skipped_fields_filled`, `skipped_manually`, `skipped_category_filter`, `skipped_classification_failed`) in `src/db/database.types.ts`.

2. **New query function** ŌĆö `src/services/extension/queries.ts`: add `fetchSkipReasons({ from, to, users, versions })`:
   - Selects the 5 new INT columns from `analytics_events` filtered by `event_type = 'catalog_batch'`, `app_source = 'tpc-extension'`, and date range.
   - Applies `.in('user_email', users)` and `.in('extension_version', versions)` ONLY when those arrays are non-empty.
   - Matches the exact filter contract used by neighboring fetchers in this file (`fetchEventVolume`, `fetchErrorRate`, etc.).

3. **New TanStack Query hook** ŌĆö `src/hooks/extension/useSkipReasons.ts`:
   - Query key: `['extension', 'skip-reasons', from, to, sortedUsers, sortedVersions]` ŌĆö sorted filters so cache key is stable.
   - Wraps `fetchSkipReasons`; aggregates the 5 columns client-side across returned rows (sums).
   - Returns `{ data: { no_photos, fields_filled, manually, category_filter, classification_failed, total }, isLoading, isError }`.

4. **New chart component** ŌĆö `src/components/extension/SkipReasonsChart.tsx`:
   - Recharts donut (PieChart) with 5 slices, one per skip reason.
   - Empty-state render when all 5 sums are 0 (covers historical periods with NULL columns and quiet periods).
   - Modelled on `ErrorRateChart.tsx` (imports, layout, error handling).
   - Uses the existing chart palette from `src/lib/chartPalette` (or extends if needed).

5. **Page insertion** ŌĆö `src/pages/Extension.tsx:157-164`: insert `<SkipReasonsChart>` section after EXT-03 ErrorRateChart, before EXT-04 PerUserTable. Same `.tpc-card p-4 mt-6` container pattern. Consumes the active date-range, user, and version filters from the existing hooks (`useDateRange`, `useUserFilter`, `useVersionFilter`).

6. **Tests**:
   - `src/components/extension/__tests__/SkipReasonsChart.test.tsx` (new) ŌĆö renders all 5 slices, empty-state, error state.
   - `src/pages/Extension.test.tsx` ŌĆö add mock + DOM assertion for new section (after EXT-03, before EXT-04).
   - `src/hooks/extension/__tests__/all-hooks-smoke.test.tsx` ŌĆö add `useSkipReasons` smoke entry.
   - `src/services/extension/__tests__/queries.test.ts` (or equivalent) ŌĆö verify `fetchSkipReasons` applies `app_source` invariant + only includes `.in()` clauses when arrays are non-empty.

Out of scope for this phase:
- New Supabase RPCs (we use direct typed column reads ŌĆö no aggregation RPC needed).
- Schema migration (owned by the cataloger phase per D-22).
- Modifying any existing chart/panel beyond the additive insertion.
</objective>
