# Phase 3 — UI Design Contract

> Visual and interaction contract for `/activity`, `/activity/sessions/:id`, and `/activity/stuck`. Generated by gsd-ui-researcher, verified by gsd-ui-checker. **Phase 3 inherits the locked token vocabulary from Phase 1 (`src/components/kit/`) and Phase 2 (`02-UI-SPEC.md`) without modification.** This document declares those tokens **by reference** and adds only the Phase-3-specific visual contract: severity tones for the Stuck Items alert, chart palettes for four new visualizations, the photo signed-URL UX, the per-card empty/error/loading copy, and the DeveloperPanel layout.

**Token inheritance source of truth:**

- **Phase 2 `02-UI-SPEC.md`** is the canonical reference for the spacing scale, typography sizes/weights, color split, table row heights, focus rings, motion-safe animation rules, and Tailwind theme tokens (`--color-accent: #2563eb`, `--color-accent-hover: #1d4ed8`).
- **Phase 1 kit** (`src/components/kit/KpiCard.tsx`, `src/components/kit/Sparkline.tsx`, `src/components/kit/PayloadViewerModal.tsx`, `src/components/kit/DateRangeFilter.tsx`) is consumed as-is; their internal token choices (notably `KpiCard`'s `text-xs font-medium` label and `text-3xl font-semibold` value) are exceptions Phase 3 inherits but does not propagate elsewhere.
- **Phase 1 components** (`src/components/EmptyState.tsx`, `src/components/ErrorState.tsx`, `src/components/TableSkeleton.tsx`, `src/components/SortIndicator.tsx`, `src/components/BackLink.tsx`, `src/components/FilterInput.tsx`) and Phase 2's `src/components/UserMultiSelect.tsx` are consumed as-is.
- **`<ErrorState>` LOCKED CONTRACT** (Phase 2 verification 2026-04-29, source: `src/components/ErrorState.tsx`): props are `{ heading: string (required); body: string (required, plain string — NOT children); onRetry: () => void (required) }`. Component renders its own internal `Retry` button. **Sibling Retry buttons are forbidden.** **Children syntax is forbidden** — body is a plain string.

**Surfaces in scope (three routes, multiple in-page surfaces each):**

1. **`/activity`** (admin surface — page composition top-to-bottom):
   - Page header with breadcrumb (`Activity`), `<DateRangeFilter>`, `<SpecialistMultiSelect>`, `<ModeToggle>`
   - **APP-01** Today KPI strip — 4 `<KpiCard>` (sessions today, items today, exports today, % AI-done today) — _right-now_, "Today's Snapshot" subheader
   - **APP-02** Active Sessions table (TanStack Table; sortable by age + specialist) — _right-now_
   - **APP-11** Stuck Items alert card — _right-now_, severity-toned, quiet success at N=0
   - **APP-03** 14-day items-per-specialist stacked bar — _fixed-window_ (always trailing 14 days)
   - **APP-04 + APP-12 paired row** — AI-status donut (left, range-driven) + House-vs-Sale split (right, range-driven)
   - **APP-05** Export Pipeline horizontal stacked bar — _range-driven_
   - **`<DeveloperPanel>`** (collapsed, dev-only via `isDevAccount(profile.email)`) — Failed-AI Breakdown + `ui_interactions` panels (Top Pages, Top Elements, Walkthrough Funnel, Recent Events Feed)

2. **`/activity` (dev surface inside the same route)** — `<DeveloperPanel>` rendered conditionally via `isDevAccount(profile?.email) ? <DeveloperPanel /> : null`. **Render-conditional, NOT `display: hidden`** — admin DOM never includes the panel (mirrors Phase 2 D-15).

3. **`/activity/sessions/:id`** (nested route — preserves `?range=`/`?specialists=`/`?mode=` URL state across back-navigation):
   - Header: `<BackLink to="/activity?{preserved-params}">Activity</BackLink>` + breadcrumb `Activity › {session.name}`
   - Session metadata card (left, 2/3 width on `xl`)
   - Photo Coverage panel (right, 1/3 width on `xl`) — **numeric only**, no thumbnail grid
   - Item list spans full width below — TanStack Table (receipt_number, title, ai_status, photo count); per-row disclosure shows lazy thumbnail strip + (dev only) Raw Item Inspector

4. **`/activity/stuck`** (nested route — bookmarkable triage page):
   - Header: `<BackLink to="/activity">Activity</BackLink>` + breadcrumb `Activity › Stuck items`
   - Single TanStack Table — receipt_number, title, ai_status, age, session.name, specialist; (dev) extra columns category, estimate, raw photo paths, "view raw item JSON"
   - Click any row → navigate to `/activity/sessions/:id` of that item's session
   - URL params on `/activity/stuck` are independent (specialist/mode filters from `/activity` are NOT inherited per D-23)

**Design north star:** `/activity` is the densest page in the v2.0 milestone — eight chart/card sections plus a collapsed dev panel. The visual rule is **calm density**: dominant white surface, secondary gray-50 only on table headers and alternating rows, **accent (`#2563eb`) reserved exclusively for the locked Phase 2 reservation list plus one new slot — the "right-now" indicator pip on the Today KPI strip header**. Severity tones (yellow / red) appear ONLY on the Stuck Items alert card. Chart series use the locked Phase 2 categorical palette family (`slate / sky / teal / amber / violet` at the `600` shade) extended only where new categorical dimensions exist (AI status, session status, mode). Failed states (red) are reserved for actual errors.

**Primary visual anchor per surface:**

- **`/activity`** — the **APP-01 Today KPI strip** is the page's first visual anchor; the **APP-03 14-day stacked bar** is the secondary anchor (mirrors Phase 2's EXT-01 placement convention). The Stuck Items alert sits between them as a **conditional anchor** — only when severity ≥ yellow does it draw the eye.
- **`/activity/sessions/:id`** — the **session metadata card + Photo Coverage panel pair** is the visual anchor; the item list is the secondary surface.
- **`/activity/stuck`** — the table itself is the only visual anchor; no chart, no KPI.

**"Right-now" indicator pip — pattern definition (NOT a new accent use):**

The Today KPI strip and Active Sessions table both display a small **green pulsing dot** (`bg-green-500 motion-safe:animate-pulse`, `w-2 h-2 rounded-full`) next to their section heading to signal "always current — ignores date range filter." Pattern is identical to Phase 2's `LiveEventFeed` running-state dot (`02-UI-SPEC.md § EXT-08 Live indicator dot`). Paired with sr-only `<span className="sr-only">Live</span>`. Stuck Items alert is also "right-now" but its severity tone carries the urgency signal — it does NOT show a green pip (avoids visual collision with the warning tone). The 14-day stacked bar (fixed-window, not "right-now") does NOT show the pip.

**Severity tone constants — Stuck Items alert card (Phase 3 NEW):**

These are the named constants the planner lifts into a constants module (`src/lib/severity.ts` or inline at top of `StuckItemsAlertCard.tsx` — plan-time decision per CONTEXT D-22 "Claude's discretion"). Tunable from a single source so card and any future re-use stay in sync.

```ts
// Stuck-items severity thresholds + tones (CONTEXT D-22).
// Tunable: revisit if operator UAT shows misclassification.
export const STUCK_ITEMS_THRESHOLDS = {
  yellowCount: 5,        // N >= 5 stuck items → yellow tone
  redAgeHours: 6,        // oldest > 6h → red tone (overrides yellow)
} as const;

// Severity tone palette — chosen at the 50/200/700 shade for low-vibration
// banner contrast (background-tinted card with deeper border + text).
// Yellow uses Tailwind 'amber' to avoid the `yellow-50` warm-cream wash that
// reads as "highlighter" rather than "warning". Amber is the same family as
// the Phase 2 spreadsheet_transform chart series (amber-600), but at 50/200/700
// these are distinct shades and a different visual role (banner vs chart bar).
export const STUCK_ITEMS_TONE = {
  none: {
    // N=0 quiet success: white card, no left border, gray icon, gray text.
    container: 'bg-white border border-gray-200',
    leftBorder: '', // no border for quiet state
    icon: 'text-gray-400',
    headline: 'text-gray-900',
    body: 'text-gray-500',
  },
  yellow: {
    // N >= 5: amber tint banner.
    container: 'bg-amber-50 border border-amber-200',
    leftBorder: 'border-l-4 border-l-amber-500',
    icon: 'text-amber-600',
    headline: 'text-amber-900',
    body: 'text-amber-800',
  },
  red: {
    // oldest > 6h: red banner. Trumps yellow.
    container: 'bg-red-50 border border-red-200',
    leftBorder: 'border-l-4 border-l-red-500',
    icon: 'text-red-600',
    headline: 'text-red-900',
    body: 'text-red-800',
  },
} as const;

// Severity classifier — pure function, called by StuckItemsAlertCard.
export function classifyStuckSeverity(args: {
  count: number;
  oldestAgeHours: number;
}): 'none' | 'yellow' | 'red' {
  if (args.count === 0) return 'none';
  if (args.oldestAgeHours > STUCK_ITEMS_THRESHOLDS.redAgeHours) return 'red';
  if (args.count >= STUCK_ITEMS_THRESHOLDS.yellowCount) return 'yellow';
  return 'yellow'; // N >= 1 but < 5 still surfaces (operator wants to see it) at the lighter tone
}
```

> **Decision note:** classifier returns `'yellow'` for `1 ≤ N < 5` rather than introducing a third "info" tone. CONTEXT D-22 specifies "yellow when N≥5, red when oldest>6h" as the start point. The classifier above is a strict superset — when `1 ≤ N < 5`, the card still shows in yellow tone because hiding the alert (or rendering it gray) at low N would underweight the signal. If operator UAT shows the lighter case is too noisy, expose a third tone constant (`info`, gray) and adjust.

> **No animation.** Per CONTEXT D-22 "Claude's Discretion: keep static; no pulsing." The right-now green pip (Today KPI strip, Active Sessions) animates via `motion-safe:animate-pulse`. The Stuck Items alert card never pulses or animates — its severity tone is enough.

**Chart palettes (Recharts) — Phase 3 introduces four new chart series mappings:**

All four palettes follow the Phase 2 chart-palette convention: `600` shade for chart marks, semantic ordering, color-blind-friendly hue separation, no accent blue. Where a category overlaps with Phase 2's mapping (e.g. `catalog_batch` → `sky-600`), Phase 3 reuses the same hex.

### APP-03 14-Day Items-per-Specialist Stacked Bar Palette

Up to N specialists per stack — **dynamic palette**. Phase 3 cannot enumerate specialists at design time.

**Allocation rule:** specialists are sorted alphabetically by `display_name` and assigned colors from a fixed cycle of 8 hues at the `600` shade. The cycle is deterministic — the same specialist always gets the same color across page reloads given the same alphabetical order.

```ts
export const SPECIALIST_COLOR_CYCLE = [
  '#0284c7', // sky-600
  '#0d9488', // teal-600
  '#7c3aed', // violet-600
  '#d97706', // amber-600
  '#4f46e5', // indigo-600
  '#db2777', // pink-600
  '#65a30d', // lime-600
  '#475569', // slate-600
] as const;

// Returns a stable color for a specialist, given the full sorted list.
export function colorForSpecialist(
  email: string,
  sortedEmails: readonly string[]
): string {
  const i = sortedEmails.indexOf(email);
  return SPECIALIST_COLOR_CYCLE[i % SPECIALIST_COLOR_CYCLE.length];
}
```

**Why `slate-600` is at the END of the cycle** (not start as in Phase 2): on this chart, the most active specialist consumes the largest visual stack area; reserving the most distinct hues for the first-listed specialists ensures a 5-specialist team gets sky / teal / violet / amber / indigo — all distinct, none gray. The 8th+ specialist falls back to slate (a calmer hue) which is acceptable because at 8+ specialists the stack is already too tall for color-by-color readability and operators rely on the legend.

Empty state (zero items in 14-day window): render `EmptyState` in the chart card body. Copy in § Per-Card Copy below.

## Severity Tone Constants

Lifted to a top-level section so the planner can extract them as a constants module without scrolling. **CONTEXT D-22 — Claude's discretion thresholds.** Tunable from one source.

```ts
// src/lib/severity.ts (proposed; planner finalizes module location)

/** Stuck-Items alert thresholds. CONTEXT D-22. Adjust here only. */
export const STUCK_ITEMS_THRESHOLDS = {
  /** Yellow when N >= this. */
  yellowCount: 5,
  /** Red when oldestAgeHours > this. Trumps yellowCount. */
  redAgeHours: 6,
} as const;

/** Tone -> Tailwind class set for the Stuck Items alert card. */
export const STUCK_ITEMS_TONE = {
  none: {
    container: 'bg-white border border-gray-200',
    leftBorder: '',
    icon: 'text-gray-400',
    headline: 'text-gray-900',
    body: 'text-gray-500',
  },
  yellow: {
    container: 'bg-amber-50 border border-amber-200',
    leftBorder: 'border-l-4 border-l-amber-500',
    icon: 'text-amber-600',
    headline: 'text-amber-900',
    body: 'text-amber-800',
  },
  red: {
    container: 'bg-red-50 border border-red-200',
    leftBorder: 'border-l-4 border-l-red-500',
    icon: 'text-red-600',
    headline: 'text-red-900',
    body: 'text-red-800',
  },
} as const;

export type StuckSeverity = keyof typeof STUCK_ITEMS_TONE;

/** Pure classifier. Test with N ∈ {0, 1, 5, 10} × age ∈ {0, 3, 6.0001, 24}. */
export function classifyStuckSeverity(args: {
  count: number;
  oldestAgeHours: number;
}): StuckSeverity {
  if (args.count === 0) return 'none';
  if (args.oldestAgeHours > STUCK_ITEMS_THRESHOLDS.redAgeHours) return 'red';
  return 'yellow';
}
```

**Test invariants:**

- `classifyStuckSeverity({ count: 0, oldestAgeHours: 999 }) === 'none'` — N=0 always wins.
- `classifyStuckSeverity({ count: 100, oldestAgeHours: 6.5 }) === 'red'` — age trumps count.
- `classifyStuckSeverity({ count: 1, oldestAgeHours: 1 }) === 'yellow'` — even N=1 surfaces as yellow.
- `classifyStuckSeverity({ count: 5, oldestAgeHours: 6 }) === 'yellow'` — exactly 6h boundary stays yellow (`>` not `>=`).
- `classifyStuckSeverity({ count: 5, oldestAgeHours: 6.0001 }) === 'red'` — fractional excess flips red.

**Re-tunable triggers:** if operator UAT shows misclassification, edit `STUCK_ITEMS_THRESHOLDS` only — no other module changes.

**No animation** — banner severity is enough. The right-now green pip animates (`motion-safe:animate-pulse`); the Stuck Items alert card never does.

## Photo Signed-URL UX Contract

The photo signed-URL story is Phase 3's most distinctive UX surface and the load-bearing piece for Success Criterion #5. Spec'd here in detail.

### Mount-time data flow (happy path)

1. Admin lands on `/activity/sessions/:id`. Session metadata + Photo Coverage panel + item list query fires. **No `createSignedUrl` calls yet.**
2. Item list renders with rows showing `photo_count` from a precomputed join (count of `photos` rows per `item_id`).
3. Admin clicks a row → row expands → for each photo on that item where `upload_status !== 'failed'`, `useSignedPhotoUrl(photo.thumbnail_path)` mounts.
4. Each thumbnail shows shimmer (`bg-gray-200 h-20 w-20 rounded animate-pulse`) until its query resolves.
5. On resolve, shimmer is replaced with `<img src={signedUrl} alt="" loading="lazy" />`.

### Lazy-disclosure interaction contract

| Interaction | Behavior |
|-------------|----------|
| Row click (non-cell area) | Toggles `isExpanded` for that row. Expanded → fetches signed URLs for that item's non-failed photos. Collapsed → query stays in cache (gcTime=10min) but the `<img>` unmounts. |
| Re-expand within 10 minutes | TanStack cache hit — thumbnails repaint instantly, no shimmer. |
| Re-expand after 10 minutes | Cache eviction → re-fetch → shimmer briefly. |
| 2-hour tab-resume | Window focus event fires → `useSignedPhotoUrl` queries with `refetchOnWindowFocus: true` re-fetch URLs (URLs older than `staleTime: 50min` are stale) → new URLs returned BEFORE the user sees a 403 → repaint without flash. **This is the Success Criterion #5 moment.** |

### Failed-photo state (CONTEXT D-13)

| Photo state | Visual treatment | `createSignedUrl` call? |
|-------------|------------------|------------------------|
| `upload_status='uploaded'` | thumbnail tile (80×80px) | **YES** (lazy on expand) |
| `upload_status='pending'` | thumbnail tile **with low-opacity overlay** (`opacity-60`) — admin sees the photo is not done uploading | **YES** (the path may exist as a stub; if 404 returned, falls back to error chip — see below) |
| `upload_status='uploading'` | thumbnail tile with overlay + spinner inset (small Heroicons `arrow-path`) | **YES** |
| `upload_status='failed'` | **inline error chip** in place of thumbnail tile: `bg-red-100 text-red-700 h-7 px-3 rounded text-sm font-semibold inline-flex items-center`. Copy: `Failed upload`. | **NO — never call.** D-13 enforced. |

**Implementation rule:** the call site checks `photo.upload_status === 'failed'` before invoking `useSignedPhotoUrl`, OR `useSignedPhotoUrl` accepts an `enabled` flag and the parent passes `enabled={photo.upload_status !== 'failed'}`. Both patterns valid; planner picks one.

### Error states for the signed-URL fetch itself

| Failure mode | Visual treatment |
|--------------|------------------|
| `useSignedPhotoUrl` query rejects (network, RLS denial, etc.) on a photo with `upload_status='uploaded'` | inline error chip: `bg-red-100 text-red-700 ...` copy `Couldn't load`. **Click to retry** — chip wrapped in a `<button>` that calls the underlying `refetch()`. `aria-label="Retry loading thumbnail"` |
| `useSignedPhotoUrl` query returns successfully but the URL itself 403s when the `<img>` actually loads | `<img onError>` handler swaps the img for the same retry chip. Same pattern as fetch-rejection. |

### Failed-photo callout in Photo Coverage panel

The Photo Coverage panel shows the upload-status breakdown numerically. When `failed > 0`:

- The "failed" row has the chip on the right side: `{N} failed` (red).
- Below the breakdown table, an additional alert paragraph: `{N} {photo|photos} couldn't upload. Check the affected items in the list below.` (singular/plural-correct.)
- The alert paragraph is `text-sm text-red-700 bg-red-50 border border-red-200 rounded p-3 mt-3` (matches the Stuck Items alert red tone family but at lower visual weight — this is informational, not actionable).

### Dev-only photo affordances

Per CONTEXT D-28:

- Under each thumbnail, a one-line caption: `font-mono text-xs text-gray-500` showing `{thumbnail_path}` (so devs can correlate with storage browser).
- A `View full-size →` text-link below the caption: opens a separate `useSignedPhotoUrl(storage_path)` query and renders the result in a `<PayloadViewerModal>` — or alternatively, a lightweight image-modal (planner picks). Either way, the storage_path call is **separate** from the thumbnail_path call (D-12 — admin only sees thumbnails). Modal title: `Full-size photo — {file basename}`.
- For `upload_status='failed'` photos, the dev also sees the `storage_path` string (gray, font-mono) under the failed-upload chip — useful for debugging the upload pipeline.

### Test invariants

- **No `createSignedUrl` call for failed photos** — Vitest unit test mocks `supabase.storage.from('photos').createSignedUrl` and asserts call count is `0` when all photos have `upload_status='failed'`.
- **Refetch-on-focus works** — Vitest behavior test mounts a row with one photo, expands it, asserts initial fetch, fires synthetic `visibilitychange` event after the URL is `> staleTime` old, asserts second fetch.
- **Cache hit on collapse + re-expand** — Vitest behavior test mounts, expands (fetch 1), collapses, re-expands within `gcTime`, asserts only 1 network fetch total.

## Accessibility Floor

Non-negotiable — the checker flags any absence. Inherits Phase 2's accessibility floor verbatim; Phase 3 adds the items below.

### Inherited from Phase 2 (verbatim)

- All interactive controls have visible focus indicators (`focus:ring-2 focus:ring-accent` or `focus-visible:ring-2`).
- Icon-only controls have `aria-label`.
- Live-indicator dots are `aria-hidden="true"`; paired sr-only `<span>Live</span>` / `<span>Paused</span>`.
- Live-feed subheading uses `<p aria-live="polite" aria-atomic="true">`.
- Sort state announced via `aria-sort="ascending" | "descending" | "none"`.
- Tables use semantic `<table>`, `<thead>`, `<tbody>`, `<th scope="col">`.
- Error-state heading has `role="alert"` (locked by `ErrorState.tsx`).
- Empty-state heading is `<h2>` semantic (locked by `EmptyState.tsx`).
- Touch targets ≥ 32px on desktop; ≥ 44px for primary table rows.
- `prefers-reduced-motion`: all animations use `motion-safe:` prefix.

### Phase 3 additions

- **Right-now indicator pip** (Today KPI strip + Active Sessions): `aria-hidden="true"` on the dot; paired sr-only `<span>Live</span>`. Pulsing animation uses `motion-safe:animate-pulse`.
- **Stuck Items alert card severity**: tone is conveyed by both color AND icon (Heroicons `exclamation-triangle` for yellow/red; `clipboard-document-list` for none). Color-blind operators see icon shape change. The headline ("{N} stuck items" vs "No stuck items") is the primary semantic; tone is reinforcement.
- **Stuck Items alert quiet state (N=0)**: container retains its `min-h-[6rem]` height — no layout reflow on every refetch (CONTEXT D-22 explicit). `aria-live="polite"` on the count text so screen readers announce when stuck items appear, but NOT on initial mount (TanStack Query handles this — first resolve doesn't trigger announcement; subsequent transitions from 0 → N do).
- **Stuck Items alert link**: when CTA `View {N} stuck items →` exists, it's a true `<Link>` (React Router) with `focus:ring-2 focus:ring-accent`. When tone is `none`, no link (no CTA). Tab order: card-as-a-region → CTA link (when present).
- **Severity-toned banner color contrast** (verified at `bg-amber-50` and `bg-red-50` per the tone constants):
  - `text-amber-900` on `bg-amber-50` = 11.2:1 (passes AAA)
  - `text-amber-800` on `bg-amber-50` = 6.9:1 (passes AAA)
  - `text-amber-600` (icon) on `bg-amber-50` = 4.6:1 (passes AA, icon-as-essential — borderline; combined with the icon's distinctive shape this passes the WCAG icon-meaning rule)
  - `text-red-900` on `bg-red-50` = 12.4:1 (passes AAA)
  - `text-red-800` on `bg-red-50` = 7.6:1 (passes AAA)
  - `text-red-600` (icon) on `bg-red-50` = 5.1:1 (passes AA)
- **Per-row disclosure trigger**: chevron is a true `<button>` with `aria-expanded={isExpanded}` and `aria-label="Expand item {receipt_number}"` / `"Collapse item {receipt_number}"`. Enter/Space toggles. Focus ring uses accent.
- **Thumbnail tiles**: `<img alt="">` (decorative for the activity-monitoring use case). The expanded-row context (`receipt_number` + `title`) provides the semantic meaning; a per-photo alt would be redundant and not contextually informative.
- **Failed-photo chip**: `role="status"` on the chip so screen readers announce the failure when a row is expanded. Click-to-retry chip on signed-URL fetch failure: `<button aria-label="Retry loading thumbnail">`.
- **Mode toggle**: implemented as `<div role="radiogroup" aria-label="Filter by session mode">` with 3 `<button role="radio" aria-checked>` children. Arrow keys navigate; Enter/Space selects. Single source of truth: URL.
- **Session metadata card**: implemented as a `<dl>` (definition list) with `<dt>`/`<dd>` pairs. Accessible to screen readers as label-value pairs.
- **Photo Coverage panel**: numeric stats use `<dl>`. The upload-status breakdown is a `<table>`. The failed callout is `<div role="alert">` when `failed > 0` (so screen readers announce on first render).
- **Stuck Items page rows**: clickable rows are `<button>` elements wrapping the cell content (TanStack Table's `getRowModel` allows this with the right cell renderer), NOT `<a>` (navigation is programmatic). `aria-label="Open session {session.name} for receipt {receipt_number}"`. Focus ring: `focus:ring-2 focus:ring-accent`.
- **Color contrast (chart palettes verified):**
  - `#0284c7` (sky-600 — processing, submitted) on `#ffffff` = 4.6:1 (passes AA for graphical objects per WCAG 1.4.11)
  - `#16a34a` (green-600 — done, exported) on `#ffffff` = 3.7:1 (passes WCAG 1.4.11 for chart marks; meets minimum-contrast for non-text)
  - `#dc2626` (red-600 — failed) on `#ffffff` = 4.5:1 (passes AA)
  - `#d97706` (amber-600 — queued, returned) on `#ffffff` = 3.6:1 (passes WCAG 1.4.11)
  - `#4f46e5` (indigo-600 — house) on `#ffffff` = 8.6:1 (passes AA)
  - `#0d9488` (teal-600 — sale, portal_upload) on `#ffffff` = 4.7:1 (passes AA)
  - All cycle palette colors at 600 shade ≥ 3.5:1 on white (passes WCAG 1.4.11 graphical-object minimum)
- **Keyboard-only journey**: filter date range → narrow by specialist → toggle mode → tab through Today KPIs (no stop — read-only) → tab to Active Sessions sortable headers → click a session → tab through metadata → tab through items → expand a row → tab to thumbnails (no stop — `<img>` not focusable) → (dev only) tab to Raw Item Inspector trigger → Escape closes any modal.

## Open Items for Plan Execution

These are the **Claude's Discretion** items from CONTEXT that the planner should resolve — not by re-asking, but by picking sane defaults during plan/executor work:

- **Stuck Items severity thresholds** — UI-SPEC commits to `yellowCount: 5`, `redAgeHours: 6` per CONTEXT D-22 starting point. Locked here as constants in `severity.ts`. Tunable from one source if UAT shows misclassification.
- **AI status donut "failed" slice visual distinction** — UI-SPEC commits to **pulled-out slice** (Recharts `<Cell>` `outerRadius` +4px on the failed cell). No heavier border, no animation. Per CONTEXT D-104 "pulled-out slice is the cleanest."
- **Pagination size** for `/activity/stuck` and Session Detail item list — UI-SPEC commits to **50 rows per page** (CONTEXT default). TanStack Table client-side pagination.
- **Active Sessions table sort default** — UI-SPEC commits to **age descending** (oldest first) per CONTEXT discretion. Specialist as secondary sort.
- **Active Sessions section right-now indicator placement** — UI-SPEC commits to **inline next to section heading**, same idiom as Phase 2 LiveEventFeed.
- **House-vs-Sale rendering** — UI-SPEC commits to **paired KPIs (NOT a pie)**. Two side-by-side count tiles inside one bordered card with a vertical divider; mode color appears as a 4px left border on each tile.
- **`<DeveloperPanel>` localStorage persistence** — UI-SPEC commits to **NOT persisting**. Page mount = collapsed. Nice-to-have deferred.
- **Walkthrough funnel step list** — UI-SPEC defers final step ordering to plan-time (planner reads TPC App's `walkthrough_step` emitter source per CONTEXT). Visual contract is locked: vertical Recharts horizontal bars, single neutral gray.
- **`get_stuck_items` shape** — out of UI-SPEC scope (data layer). The card and page consume the same RPC result; planner picks one-RPC vs paired based on row volume.
- **`/activity` nav icon** — UI-SPEC commits to **Heroicons `clipboard-document-list`** (outline, stroke-width 1.5).
- **URL filter param naming** — UI-SPEC commits to **comma-separated single-key form** matching Phase 1 / Phase 2 precedent: `?specialists=email1,email2`, `?mode=house|sale|all`.
- **Right-now indicator on Stuck Items alert** — UI-SPEC commits to **NOT showing** the green pip on the alert card (severity tone carries the urgency signal; double-signaling muddles it). The Today KPI strip and Active Sessions table DO show the pip.
