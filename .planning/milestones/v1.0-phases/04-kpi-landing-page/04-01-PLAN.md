Shape the RPC MUST return (JSONB):
```json
{
  "current":  { "revenue": 0, "sell_through": null, "lots_sold": 0, "sales_count": 0 },
  "previous": { "revenue": 0, "sell_through": null, "lots_sold": 0, "sales_count": 0 }
}
```

- `revenue` → `COALESCE(SUM(net_revenue), 0)` — never null
- `sell_through` → `SUM(lots_sold)::numeric / NULLIF(SUM(lots_auctioned), 0)` — null when denominator is 0
- `lots_sold` → `COALESCE(SUM(lots_sold), 0)` — never null
- `sales_count` → `COUNT(*)` — never null (0 when empty)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → Supabase PostgREST | Authenticated user calls `supabase.rpc('kpi_summary', ...)`; PostgREST dispatches to Postgres via the `authenticated` role |
| PostgREST → Postgres (security-definer RPC) | The RPC executes as the function owner (postgres); inputs are typed `date` — no string path into the SQL query body |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Elevation | kpi_summary RPC (search-path hijack on security-definer) | mitigate | `set search_path = public, pg_temp` on the function. Verified by grep in Task 1 acceptance. |
| T-04-02 | Tampering | kpi_summary RPC (SQL injection via date params) | mitigate | All 4 parameters typed `date`; no string concat in body; Postgres casts raise on malformed input (blocks "2026'; drop table sales; --"). |
| T-04-03 | Elevation | kpi_summary RPC (client RBAC bypass by non-admin) | mitigate | Belt: RLS on `public.sales` is admin-only; suspenders: explicit `private.is_admin(auth.uid())` gate at RPC entry raises `Access denied` for non-admins. |
| T-04-04 | Information Disclosure | kpi_summary RPC (privilege escalation via PUBLIC grant) | mitigate | `revoke all on function ... from public` + `grant execute ... to authenticated` (only). `anon` and `public` cannot execute. |
</threat_model>
