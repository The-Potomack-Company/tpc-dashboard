## RPC Return Shapes (sample payloads)

**public.department_rankings(range_start, range_end) → jsonb array:**

```json
[
  {
    "department_code": "ASN",
    "display_name": "Asian art",
    "sales_count": 5,
    "total_revenue": 123456.78,
    "avg_sell_through": 0.68,
    "lots_above_estimate": 12
  }
]
```

Sorted DESC by `total_revenue`. Wrapped in `coalesce(..., '[]'::jsonb)` so no-result returns `[]`.

**public.department_revenue_series(range_start, range_end, dept_codes) → jsonb array:**

```json
[
  { "sale_date": "2024-01-15", "sale_number": "2024-0001", "ASN": 1000, "FRN": 500 },
  { "sale_date": "2024-02-19", "sale_number": "2024-0002", "ASN": 800,  "FRN": 450 }
]
```

Wide rows sorted ASC by `sale_date`. One key per entry in `dept_codes`. INNER JOIN drops sales with no matching dept-rows (empty-point collapse).

**public.department_share_series(range_start, range_end, top_n) → jsonb object:**

```json
{
  "rows": [
    { "sale_date": "2024-01-15", "sale_number": "2024-0001", "ASN": 0.4, "FRN": 0.3, "other": 0.3 }
  ],
  "top_codes": ["ASN", "FRN"]
}
```

Object envelope (not array). `'other'` key aggregates non-top-N shares. Shares in [0,1]; `nullif(total, 0)` guards divide-by-zero.

## Decisions Made

- **Zero-arg admin gate** — all three RPCs call `private.is_admin()` (not `private.is_admin(auth.uid())` as the plan instructed) because the Phase 1 helper is zero-arg. The T-06-01-03 mitigation (admin gate inside function body) is present; only the call signature differs from the plan's literal text.
- **Any missing sale_number → fully invalid** on `useSalesComparison` — throwing with the missing list surfaces uniformly as the invalid-URL card (06-RESEARCH Open Question #3).
- **Empty deptCodes → disabled fetch** on `useDepartmentRevenueSeries` — the RPC's `cardinality=0` branch means "all depts" server-side, but at the client an empty chip-bar is "show nothing." Keeping the fetch off the wire is more literal to the UI state.
- **Envelope (not array)** for `department_share_series` return — returning both `rows` and `top_codes` in one round-trip means the hook doesn't need a second request just to establish the legend order.
- **Nested freeze on EMPTY_SHARE** — outer envelope + inner rows + topCodes are all frozen so consumers can't mutate any layer in place.
