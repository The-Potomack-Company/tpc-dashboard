# Phase 3 — UI Design Contract

> Visual and interaction contract for Sale Views. Generated by gsd-ui-researcher, verified by gsd-ui-checker. Extends Phase 1's locked tokens — adds table, filter-input, skeleton, and validation-banner contracts without introducing a new visual vocabulary.

**Surfaces in scope:**
1. `/sales` — sales list page (dense sortable + filterable TanStack Table, 457 rows, client-rendered with virtualization)
2. `/sales/:saleNumber` — sale detail page (optional validation-warning banner + summary KPI card + department breakdown table)
3. `/sales/:saleNumber` (404 state) — "Sale not found" screen with back link

**Design north star:** Keep Phase 1's visual vocabulary intact. Data tables are the first complex UI in this project — they lean on gray tones (not accent) to stay readable across hundreds of rows. Accent remains reserved for Phase 1's 5 elements plus exactly two Phase 3 additions (active sort column indicator, row focus ring). Desktop-first per INFR-03 — sidebar collapses to an icon rail at `md` (768–1023px); below 768px is out of scope.

**Primary visual anchor per surface:**
- `/sales` — the sales list table, spanning the full content width below the heading (heading and filter input frame it but do not compete for visual weight).
- `/sales/:saleNumber` — the summary KPI card sits directly below the heading as the first-fold focal element; the department breakdown table is the secondary anchor below.

**Size rule (unchanged from Phase 1):** exactly four sizes. The `text-xs` (12px) in Phase 1 sidebar (`v0.0.0` footer, "Coming soon" aside) is a *Phase 1* exception carried forward; Phase 3 does not introduce any new `text-xs` usage.

**Numeric formatting conventions** (applied in cell renderers, not CSS):

| Value type | Format | Example | Implementation |
|-----------|--------|---------|----------------|
| Currency (USD) | `$X,XXX,XXX.XX` | `$1,234,567.89` | `new Intl.NumberFormat('en-US', { style: 'currency', currency: 'USD' }).format(value)` |
| Percentage | `XX.X%` (one decimal) | `68.4%` | `new Intl.NumberFormat('en-US', { style: 'percent', minimumFractionDigits: 1, maximumFractionDigits: 1 }).format(rawRatio)` |
| Integer count (lots, bidders) | `X,XXX` | `1,247` | `new Intl.NumberFormat('en-US').format(value)` |
| Date | `Mon D, YYYY` | `Nov 16, 2022` | `new Intl.DateTimeFormat('en-US', { month: 'short', day: 'numeric', year: 'numeric' }).format(date)` |
| Estimate range | `$low – $high` (en-dash, non-breaking spaces) | `$500,000 – $800,000` | Two `Intl.NumberFormat` calls joined by ` – ` (U+2013) |
| Null / missing | em-dash | `—` (U+2014) | Single shared constant; never `"N/A"`, never empty cell |

These conventions live in a `src/lib/format.ts` module exporting `formatCurrency`, `formatPercent`, `formatCount`, `formatDate`, `formatEstimateRange`, and the `EMPTY` constant. Executor creates this file; planner should include it as a task.

**Intentional consistency exception — `Retry` (single-verb CTA):** Every retry surface in Phase 3 (sales list error, sale detail error, any future retry) uses the single word `Retry`. ## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Sales list: column header click | Cycles sort state (`unsorted` → `asc` → `desc` → `unsorted`) via TanStack Table `onSortingChange`. Only one column sorted at a time (single-column sort, `enableMultiSort: false`). Default sort is `sale_date DESC`; clicking the Date header a third time reverts to this default, not to truly "unsorted". |
| Sales list: sort indicator | Inactive columns: `chevron-up-down` icon in `text-gray-400` (16×16). Active asc: `chevron-up` in `text-accent`. Active desc: `chevron-down` in `text-accent`. Icon sits to the right of the header label with `gap-1` (4px). |
| Sales list: filter input typing | Debounced 200ms. Filters across `title` + `sale_number` (case-insensitive, `String.prototype.toLowerCase().includes()`). Clears filter via `Esc` keypress or a trailing `×` button inside the input (aria-label: `Clear filter`). |
| Sales list: row click | Navigates to `/sales/:sale_number` via `react-router` `<Link>` wrapping the row content. Row has `role="link"` and `cursor-pointer`. Hover surfaces `bg-gray-100 dark:bg-gray-800`. |
| Sales list: row keyboard | Rows are focusable (`tabIndex={0}`). Enter or Space fires the same navigation. Focus ring uses accent (see Color § accent reservation #7). |
| Sales list: virtualization | TanStack Virtual `useVirtualizer({ count, getScrollElement, estimateSize: () => 44, overscan: 10 })`. Scroll container is the table's surrounding `<div>` with `overflow-y-auto`, `max-h-[calc(100dvh-16rem)]` (viewport minus shell header + page heading). Sticky table header inside this container (`sticky top-0 bg-gray-50 dark:bg-gray-800 z-[1]`). |
| Sale detail: table row interaction | Department table rows are **not** clickable — they have no focus ring, no hover state, no cursor change. Read-only rows. |
| Sale detail: department table sort | Same sort-indicator behavior as sales list, default `revenue DESC`. |
| Sale detail: validation banner action | `Reload sale` is a button (not a link) that calls `queryClient.invalidateQueries({ queryKey: ['sale', saleNumber] })`. Button has `focus:ring-2 focus:ring-accent` (accent reservation #2 — pre-existing). |
| Sale detail: back link | Standard react-router `<Link to="/sales">`; same focus ring pattern. |
| Input focus (filter input, buttons) | `focus:ring-2 focus:ring-accent` + `focus:border-accent` — identical to Phase 1 Login inputs. |
| Error state: retry | Button calls `refetch()` from the relevant TanStack Query hook. Shows spinner inline for the duration of the retry (same border-spinner pattern as Phase 1 auth loading). |

**Loading patterns:**
- **Sales list initial load** — render the page shell (heading, subtitle skeleton, filter input placeholder-style, and 10 shimmer rows matching column widths). Shimmer uses `animate-pulse` on `bg-gray-200 dark:bg-gray-700` `h-4` bars inside each cell.
- **Sale detail initial load** — render the page shell, a 4×3 grid of skeleton KPI tiles (each tile has a label-sized `h-3` bar and a value-sized `h-4` bar), and a 6-row skeleton department table below.
- **Refetch after success** — do NOT replace the rendered data with skeletons. TanStack Query `keepPreviousData` behavior + a subtle 2px top border-loading indicator on the page header (reserved micro-interaction; not a spinner) signals activity without clobbering the UI. If a refetch fails while data is already rendered, show a toast-style inline error banner above the table. (This is the only toast-adjacent surface in the project; it's a single React component, not a toast library. Revisit at Phase 7.)
- **Per-row loading** — none in Phase 3. Rows are atomic — they either render or skeleton-render.

**Error patterns:**
- Inline error states only. No global error dialog, no toast library.
- Every error surface has `role="alert"` on its heading.
- Retry is always available and always labelled `Retry` — no "Try again", no "Refresh" — one verb. See the Intentional consistency exception in Copywriting.

**Keyboard tab order on `/sales`:** header user menu (inherited shell) → filter input → clear-filter button (if visible) → first column header → next header → ... → first data row → next row → ... (virtualization preserves tab order for the rows currently in the DOM; overscan of 10 avoids lost focus on scroll).

**Keyboard tab order on `/sales/:saleNumber`:** header user menu → back link → validation banner button (if present) → department table first column header → ... (skip the summary card — its content is non-interactive).

### Responsive breakpoints

- **Default (desktop, 1024px+):** Full sidebar (240px), full table widths, KPI grid 4 columns.
- **`md` (768–1023px):** Sidebar collapses to an icon rail (64px wide, icons only — Phase 3 adds this because Phase 1's decision was "links disabled"; now `/sales` is active, the icon rail needs a Sales icon from Heroicons, `table-cells`). Header shrinks to show only the avatar + menu. Tables remain at full width; horizontal scroll appears if columns can't fit (set `overflow-x-auto` on the scroll container). KPI grid drops to 3 columns.
- **Below 768px:** Out of scope (INFR-03). Do not add any small-viewport optimizations.

**Sidebar icon-rail details** (new in Phase 3, implements the Phase 1 "collapses on tablet per existing pattern" note):
- At `md:`, `aside` becomes `w-16` (64px) and each nav item becomes a 44×44 square with centered icon + `aria-label`. Tooltip on hover surfaces the label — use the native `title` attribute (no custom tooltip library).
- Heroicons used: `Sales = table-cells`, `Trends = chart-bar`, `Departments = building-library`, `Team = users`, `Reports = document-text`, `Custom Charts = chart-pie`. All outline, stroke-width 1.5.

## Accessibility Floor

Non-negotiable — the checker flags any absence.

- All interactive controls (filter input, sort buttons, row links, retry button, banner button, back link) have visible focus indicators via `focus:ring-2 focus:ring-accent` (inputs) or `focus-visible:ring-2 focus-visible:ring-accent` (rows).
- Every icon-only control has an `aria-label` (clear-filter button: `Clear filter`; sidebar icon-rail links: link label).
- Filter match count live region: `<p aria-live="polite" aria-atomic="true">{n} of {total} sales</p>` so screen readers announce result count changes.
- Sort state announced: each sortable column header has `aria-sort="ascending" | "descending" | "none"` that updates with state changes.
- Validation banner uses `role="alert"`; the container, not the text inside.
- Error-state headings use `role="alert"`.
- Tables use semantic `<table>`, `<thead>`, `<tbody>`, `<tfoot>`, `<th scope="col">`. No `<div>`-tables.
- Row links (`/sales` rows) are true `<a>` tags (via react-router `<Link>`) wrapping the row content — clicking anywhere in the row navigates; keyboard Tab lands on the link; Enter activates.
- Touch targets: table row height 44px (≥ WCAG 24px minimum and ≥ 44px for clickable rows). Filter input 40px (non-primary control; WCAG 24px floor satisfied).
- Color contrast: all text on `bg-gray-50` (table header) passes 4.5:1 — `gray-700` on `gray-50` = 9.2:1, `gray-500` on `gray-50` = 4.6:1. Amber banner text `amber-900` on `amber-50` = 10.4:1. Active sort accent (`#2563eb`) on `gray-50` = 4.5:1.
- `prefers-reduced-motion`: shimmer animation (`animate-pulse`) is suppressed via `motion-safe:animate-pulse`. TanStack Virtual scroll is passive — no motion concerns.
- Numeric cells using `tabular-nums` aid scan accuracy for users with low vision by aligning digit columns.
- Keyboard-only users can complete all Phase 3 journeys: filter → find sale → Tab into rows → Enter to navigate → browse detail → Enter on Reload if banner present → Back link to return.
