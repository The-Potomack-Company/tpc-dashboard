# Phase 2 — UI Design Contract

> Visual and interaction contract for `/extension`. Generated by gsd-ui-researcher, verified by gsd-ui-checker. Extends the locked tokens established by Phase 1's UI kit (`src/components/kit/`) and v1.0 Phase 3 (`03-UI-SPEC.md`, retained as the canonical token reference). No new visual vocabulary is introduced.

**Surfaces in scope (single route, two surfaces inside it):**

1. `/extension` (admin surface — always rendered when not in empty-gate state)
   - Page header with `<DateRangeFilter>` + user multi-select filter
   - **EXT-01** 14-day stacked bar chart (event volume by `event_type`, 5 series)
   - **EXT-02** 5 KPI cards (count + previous-period delta + sparkline) — one per event type
   - **EXT-03** Horizontal bar chart of error rate per event type
   - **EXT-04** Per-user table (TanStack Table — sortable; `Unknown` bucket for `user_email IS NULL`)
   - **EXT-05** Recent Errors table (TanStack Table — newest 100; row click is no-op for admin)
   - **EXT-08** Live event feed (TanStack `refetchInterval: 10_000`; Pause / Resume button; admin row click is no-op)
2. `/extension` (developer surface — collapsed `<DeveloperPanel>` rendered only when `isDevAccount(profile.email)` is true)
   - **EXT-09** `extension_version` multi-select + dominant-version badge
   - **EXT-10** Two cancellation-rate KPI cards (W2 `catalog_batch`, W3 `portal_upload`)
   - **EXT-06** Payload-viewer trigger affordance on EXT-05 / EXT-08 rows (admin sees row, dev sees a clickable affordance opening `<PayloadViewerModal>`)
3. `/extension` (full-page empty gate — when lifetime `analytics_events.app_source = 'tpc-extension'` rows = 0)
   - Single centered `<EmptyState>`; no charts, tables, or feed mounted

**Design north star:** The page is dense with read-only diagnostic data. Charts and tables share the same gray-on-white surface vocabulary that Phase 1 established. **Accent (`#2563eb`) is reserved** — chart series and KPI deltas use semantic palette colors (red for errors, neutral grays for non-error series). The dev panel must be visually quiet: it sits at the bottom, collapsed by default, and uses the same surface treatment as everything above it (no inverted card, no stripe, no warning band) so an admin's eye does not land on it.

**Primary visual anchor per surface:**

- **`/extension`** (admin) — the **EXT-01 14-day stacked bar chart** spans full content width directly below the filter row; it is the page's single hero element. The 5 KPI cards row sits below it as the secondary anchor.
- **`/extension`** (dev panel expanded) — the dominant-version badge (right-aligned next to the panel title) and the two cancellation-rate KPI tiles share visual weight; neither dominates.
- **`/extension`** (empty gate) — the centered `<EmptyState>` card with the heading "No extension events yet" is the only element on screen.

| Role | Value | Usage |

**Accent reserved for** (Phase 1 + v1.0 + Phase 2 — total 7; Phase 2 introduces **zero new** accent uses):

2. *(Phase 1)* Input focus rings + focus border
3. *(Phase 1)* Active sidebar link left-edge indicator + text — **Phase 2 activates** this for `/extension` (the route's NavLink becomes the first active sidebar entry)
4. *(Phase 1)* Header user avatar badge fill
5. *(Phase 1)* Full-page auth-loading spinner
6. *(v1.0 Phase 3)* Active sort column indicator (`SortIndicator.tsx`) — **Phase 2 reuses** for EXT-04 / EXT-05 sortable headers
7. *(v1.0 Phase 3)* Table row focus ring — **Phase 2 reuses** if EXT-04 / EXT-05 rows become click-to-navigate (Phase 2 they are not, so this slot is dormant for now)

Accent is **NOT** used for:
- Chart bar colors (use the role-based palette below)
- KPI card values, deltas, or sparklines (KPI delta semantic palette is green/red/gray; sparklines use `currentColor` from a parent gray text class)
- Pause/Resume button background (secondary outline style — gray border + gray text + accent focus ring)
- Live-feed event-type badges (gray-tone palette below)
- DeveloperPanel collapse/expand chevron (gray)
- Empty-state CTA (no CTA in empty state — the page just waits for v2.0 extension to ship)

**Chart palette (Recharts):**

EXT-01 stacked bar uses **5 distinguishable hues** from Tailwind's default palette — neutral, color-blind-friendly, non-accent. **Series-to-color mapping is locked here** so executor cannot coin their own:

| Event type | Tailwind | Hex | Notes |
|------------|----------|-----|-------|
| `catalog_single` | `slate-500` | `#64748b` | The most frequent event; calmest hue — does not dominate |
| `catalog_batch` | `sky-600` | `#0284c7` | W2 workflow; cool blue. **Distinct from `--color-accent` (`#2563eb`)** — sky vs. blue, deliberately not the accent token |
| `portal_upload` | `teal-600` | `#0d9488` | W3 workflow; cool green |
| `spreadsheet_transform` | `amber-600` | `#d97706` | W4 workflow; warm amber (echoes the v1.0 Phase 3 validation banner palette family but at the saturated 600 shade — different role, different shade, no conflict) |
| `data_import` | `violet-600` | `#7c3aed` | Lowest-volume event; distinct hue avoids being lost in the cool palette |

**Why these:** They are 5 hues separated around the wheel, all at the `600` shade for consistent saturation, none of them are the accent blue, none are red (red is reserved for error signals).

EXT-03 error-rate horizontal bar uses **a single neutral color** (`gray-400`, `#9ca3af`) for all bars regardless of event type, with the bar background shaded `gray-100`. Rationale: error rate is a single metric type per row; using 5 different colors would imply meaning where there is none. The error-rate value text ≥ 5% switches to `text-red-600` to flag attention without a bespoke chart palette.

**KPI delta colors** (locked by `KpiCard.tsx`, Phase 1):
- `direction: 'up'` → `text-green-600`
- `direction: 'down'` → `text-red-600`
- `direction: 'flat'` → `text-gray-500`

**Caller (KpiStrip.tsx) chooses semantic direction**:
- For `catalog_single`, `catalog_batch`, `portal_upload`, `spreadsheet_transform`, `data_import` count KPIs → **more is good**; pass `up` for increase, `down` for decrease.
- For dev-panel cancellation-rate KPIs → **higher is bad**; flip the direction (decrease in cancellation-rate uses `up` (good = green), increase uses `down` (bad = red)).

**Live-feed event-type badge palette (per row):**

The badge background uses the chart palette hue at `100` shade, text at `700` shade. This is the only place where chart series colors leak into chip styling, and it does so consistently:

| Event type | Background | Text |
|------------|-----------|------|
| `catalog_single` | `bg-slate-100` | `text-slate-700` |
| `catalog_batch` | `bg-sky-100` | `text-sky-700` |
| `portal_upload` | `bg-teal-100` | `text-teal-700` |
| `spreadsheet_transform` | `bg-amber-100` | `text-amber-800` (amber-700 fails contrast on amber-100) |
| `data_import` | `bg-violet-100` | `text-violet-700` |

**Error indicator on live-feed rows:** Rows where `error_message IS NOT NULL` get a left border (`border-l-2 border-red-500`) and the timestamp text turns `text-red-600`. The badge itself does NOT change color — the error signal is the border + timestamp, the event-type semantic stays consistent. This pattern is also used on Recent Errors table rows (every row has an error by definition; the left border treatment is implicit on the entire table, see Layout).

**Borders and radius** (unchanged from Phase 1): `rounded-lg` (8px) for KPI cards, chart cards, DeveloperPanel, payload modal. Tables remain unrounded inside their wrapping rounded-card containers (matches v1.0 Phase 3 pattern).

| EXT-05 column header click | Cycles sort state — default `created_at DESC`. |
| EXT-08 live feed: row click | Same admin-vs-dev split as EXT-05. |

**Loading patterns:**

- **First mount, empty-gate probe in flight** → page shell + filter row + a single centered spinner OR per-card skeletons; **NOT** the full-page empty state (gate hasn't resolved yet). Pattern: render the filter row immediately; render skeleton cards/charts; when `useExtensionGate` resolves to `isEmpty: true`, replace everything below the filter row with the empty state.
- **Per-card initial load** → each chart/table/feed shows its own skeleton (chart-card skeleton = gray pulsing rectangle filling the card body; table skeleton = `<TableSkeleton>` with column-width hints; KPI card skeleton = built into `KpiCard loading` prop).
- **Refetch after success** → do NOT replace rendered data with skeletons. TanStack Query default behavior preserves previous data. No top-of-page loading bar (Phase 2 is calm; the live-feed live indicator is the only "something is moving" affordance).
- **Empty state (per-card, lifetime ≠ 0)** → render `<EmptyState>` with the per-section copy from the Copywriting Contract. Single emphasis: do NOT render an empty chart with no bars; replace the chart card body with the empty-state component.

**Error patterns:**

- Per-card inline error states only (D-21). No global error dialog, no toast library.
- Every error heading has `role="alert"` (already locked by `ErrorState.tsx`).
- Retry button always available, always labeled `Retry`. The button's spinner-during-retry pattern is owned by the underlying TanStack Query refetch lifecycle — `<ErrorState>` does not animate.

1. Header user menu (inherited shell)

## Accessibility Floor

Non-negotiable — the checker flags any absence.

- All interactive controls (date range buttons, user multi-select trigger, sort headers, Pause/Resume button, retry buttons, DeveloperPanel toggle, `View →` cells, payload modal buttons) have visible focus indicators via `focus:ring-2 focus:ring-accent` (or `focus-visible:ring-2` for items where the resting state should not show a ring on click).
- Every icon-only control has an `aria-label` (Pause: `Pause live feed`; Resume: `Resume live feed`; clear-filter inside `<UserMultiSelect>`: `Clear user filter`; DeveloperPanel toggle: per-state `Expand`/`Collapse developer panel`).
- Live feed live-indicator dot is `aria-hidden="true"` (visual cue only); paired sr-only `<span>Live</span>` / `<span>Paused</span>` carries the meaning.
- Live feed section subheading (`Tailing latest 50 events · refreshes every 10s` / `Paused · {n} events shown at pause time`) uses `<p aria-live="polite" aria-atomic="true">` so screen readers announce state changes when Pause/Resume is toggled.
- Sort state announced: each sortable column header has `aria-sort="ascending" | "descending" | "none"` updating on state change (continues v1.0 Phase 3 contract).
- Validation banner pattern not used in Phase 2 (no schema-drift warnings on this page). Error states use `role="alert"` on the heading (locked by `ErrorState.tsx`).
- Empty-state heading is `<h2>` semantic, not `<div>` (locked by `EmptyState.tsx`).
- Tables use semantic `<table>`, `<thead>`, `<tbody>`, `<th scope="col">`. No `<div>`-tables.
- `View →` action cells in EXT-05 / EXT-08 (dev only) are true `<button>` elements, not anchors — they trigger a modal, not a navigation. They have `aria-haspopup="dialog"` so screen readers announce the modal relationship.
- Payload modal accessibility is owned by native `<dialog>` (`PayloadViewerModal.tsx`) — focus trap and Escape close are handled by the platform.
- Touch targets: header user menu button (inherited 44px), date range preset buttons (32px height — slightly under 44px; acceptable for trackpad/mouse desktop tool per INFR-03 desktop-only scope; matches `DateRangeFilter.tsx` Phase 1 spec), Pause button (32px height — same rationale), table row (44px), live-feed row (40px).
- Color contrast (verified at `bg-white` and `bg-gray-50`):
  - `text-gray-700` on `bg-white` = 12.6:1 (passes AAA)
  - `text-gray-500` on `bg-white` = 5.7:1 (passes AA)
  - `text-red-600` on `bg-white` = 5.9:1 (passes AA)
  - `text-green-600` on `bg-white` = 4.8:1 (passes AA, KPI delta)
  - `text-amber-800` on `bg-amber-100` = 4.8:1 (passes AA, spreadsheet_transform badge — using `amber-800` text not `amber-700` because `amber-700`/`amber-100` is 4.0:1, fails AA)
  - All other badge text-on-bg combinations (slate, sky, teal, violet at 700-on-100) ≥ 4.6:1 (verified via WCAG calculator on the Tailwind palette)
  - Active sort accent (`#2563eb`) on `bg-gray-50` = 4.5:1 (matches v1.0 Phase 3 verification; meets minimum)
- `prefers-reduced-motion`: live-feed dot pulse uses `motion-safe:animate-pulse`; chevron rotation in DeveloperPanel uses `motion-safe:transition-transform`. Skeleton shimmer continues v1.0 Phase 3's `motion-safe:animate-pulse` (locked by `TableSkeleton.tsx`).
- Numeric cells (per-user counts, error counts, error rates, KPI values) use `tabular-nums` for digit alignment.
- Keyboard-only users can complete all Phase 2 journeys: filter date range → narrow by user → scan KPIs → sort per-user table → tab into recent errors → (dev only) View → payload → Escape modal → Tab to live feed → Pause/Resume → (dev only) toggle DeveloperPanel.
