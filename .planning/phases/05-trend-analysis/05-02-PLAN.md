| props → DOM | Title strings, preset labels are all literal constants in component (no external string interpolation) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02-XSS | Tampering | DateRangeFilter status line | mitigate | `formatDate` is the only user-derived string; it returns a formatted date literal from `Intl.DateTimeFormat` — no HTML ever reaches the DOM (React auto-escapes text children). The inline alert text is a constant. |
| T-05-02-INPUT | Tampering | DateRangeFilter Apply | mitigate | Apply validates start ≤ end via lexicographic `yyyy-mm-dd` comparison; invalid ranges are rejected with `role="alert"` and onChange never fires. Downstream hook in plan 05-03 will additionally validate the string shape before embedding into a Supabase query (defense in depth). |
| T-05-02-RBAC | Information Disclosure | (none) | accept | No data fetching in plan 05-02 — pure UI controls. RBAC enforced by the hooks in plan 05-03 + RLS. |
| T-05-02-A11Y | Usability | DateRangeFilter, MetricToggle | mitigate | WAI-ARIA radiogroup pattern verified in test suite (aria-checked, aria-label, aria-expanded on Custom, role=alert on validation error). Focus ring uses accent reservation #2 and #7. Touch targets 40px (WCAG 24px min). |
| T-05-02-DESTRUCTIVE | Elevation of Privilege | (none) | accept | Read-only controls emitting values via onChange. No destructive actions. |
