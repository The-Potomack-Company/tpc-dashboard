```ts
// src/hooks/useSales.ts
const EMPTY_SALES: readonly Sale[] = Object.freeze([]) as readonly Sale[];
export function useSales() {
  return useQuery<Sale[]>({
    queryKey: ['sales'],
    staleTime: 5 * 60_000,
    queryFn: async () => {
      const { data, error } = await supabase.from('sales').select('*')
        .order('sale_date', { ascending: false, nullsFirst: false });
      if (error) throw error;
      return data ?? (EMPTY_SALES as Sale[]);
    },
  });
}
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Range props → Supabase | `range.start` / `range.end` are user-controlled (via DateRangeFilter) and embedded into PostgREST filter predicates |
| Supabase → hook caller | Data returned is trusted to match the generated Database types (Phase 1 RLS) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03-XSS | Tampering | (none — hooks don't render) | accept | Hooks return raw data; consumers (plans 05-04..07) render via React text children which auto-escape. |
| T-05-03-INJ | Tampering | useSalesInRange, useDepartmentGrid | mitigate | `range.start/end` are strings but the supabase-js client parameterizes `.gte('sale_date', value)` — the column name is a literal and the value is passed as a parameter, not concatenated into raw SQL. Additionally, plan 05-02 validates the shape in Apply-button handler. |
| T-05-03-RBAC | Information Disclosure | both hooks | mitigate | Phase 1 RLS policy on `sales` + `sale_departments` is admin-only SELECT. The hooks rely on that policy — they do NOT check `useAuthStore().isAdmin` client-side (client checks are advisory per Phase 1 rationale). A non-admin user with stale cookies hitting this hook gets an RLS-filtered empty array or auth error; the Trends page (plan 05-07) renders the resulting empty state gracefully. |
| T-05-03-A11Y | Usability | (none — hooks are headless) | accept | Accessibility belongs to the consuming components. |
| T-05-03-DESTRUCTIVE | Elevation of Privilege | (none) | accept | Both hooks are SELECT-only; no INSERT/UPDATE/DELETE calls. |
</threat_model>
