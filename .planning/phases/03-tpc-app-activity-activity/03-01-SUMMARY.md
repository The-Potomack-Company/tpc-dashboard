## Decisions Made

1. **`get_export_pipeline` includes `'completed'` (Open Q1 locked: yes — 5 segments).** TPC App migration `20260320000000_add_completed_status.sql` extended the status enum; prod data uses it. Excluding it would render bars that don't sum to total session count.
2. **`get_walkthrough_funnel` step list = canonical TPC App emitter list (14 steps).** Verified at execute time from `~/Projects/TPC_App/TPC_App/src/components/walkthrough/walkthroughSteps.tsx` (SHARED_STEPS 1-8 + ADMIN_STEPS 9-12 + SPECIALIST_STEPS 13-14). No placeholder needed.
3. **`get_stuck_items` extends RESEARCH Pattern 2 with 3 dev-surface columns** (`category`, `estimate`, `photo_paths text[]`) per D-28. Admin UI hides them; dev UI surfaces them in `<RawItemInspector>`. `photo_paths` filters to `upload_status = 'failed'` to surface broken thumbnails for diagnosis (D-13).
4. **`get_failed_ai_breakdown` uses long-form output** with a `dimension` discriminator column. One RPC + 3 GROUP BY blocks UNION ALLed, instead of three separate RPCs over the same scoped slice. Saves 2 roundtrips on every refresh of the failed-AI breakdown panel.
5. **`get_house_sale_split` always returns both rows** (house, sale) regardless of `p_mode`. The UI is responsible for hiding the unselected tile. This keeps the contract symmetric with the donut/distribution RPCs.
6. **Verifier comment-stripping is mandatory** before counting. The migration's header invariant block discusses every D-XX in plain English; without stripping, the verifier would false-positive on its own documentation. Phase 2's `verify-extension-app-source-scope.mjs` line 43-46 idiom is copied verbatim.
7. **Verifier load is announced in OK banners** (e.g., `OK — 11 canonical "sessions.mode = p_mode" filters`). Future engineers reading CI logs see the invariant strength immediately, not just a green check.
8. **`package.json` prebuild wiring is delegated to Plan 03-02** (Wave 1 coordination). If both 03-01 and 03-02 edited the same `package.json` field in parallel worktrees, the second writer would clobber the first. Plan 03-02 declares `depends_on: [03-01]` to serialize after this plan.
