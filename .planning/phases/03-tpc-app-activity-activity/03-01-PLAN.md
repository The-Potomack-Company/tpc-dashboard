2. **Function 1 — `get_today_kpis(p_specialists text[] default array[]::text[], p_mode text default 'all')`** — right-now per D-14. Returns ten counts in a single row: today + yesterday for each of (sessions, items, exports, items_done, items_total — the last two split so callers can compute `items_done_today / items_total_today` for the % AI-done KPI). Today bounds via `date_trunc('day', now(), 'America/New_York')` (start of today) and `date_trunc('day', now(), 'America/New_York') + interval '1 day'` (end of today). Yesterday is the prior 24h: subtract `interval '1 day'` from both bounds.

  Inside the function body, define a `bounds` CTE:
  ```sql
  with bounds as (
    select
      date_trunc('day', now(), 'America/New_York')                     as today_from,
      date_trunc('day', now(), 'America/New_York') + interval '1 day'  as today_to,
      date_trunc('day', now(), 'America/New_York') - interval '1 day'  as yday_from,
      date_trunc('day', now(), 'America/New_York')                     as yday_to
  ), specialist_ids as (
    select id from public.profiles
     where role = 'specialist' and is_active = true
       and (cardinality(p_specialists) = 0 or email = any(p_specialists))
  )
  ```
  Then five paired aggregates (today + yesterday) for sessions, items, items_done, items_total, exports. Sessions count via `public.sessions s` filtered by `created_at` between bounds, `(p_mode = 'all' or s.mode = p_mode)`, and `(cardinality(p_specialists) = 0 or s.assigned_to in (select id from specialist_ids))`. Items count via `public.items i join public.sessions s on s.id = i.session_id` with mode filter on `s.mode` (D-20) and specialist filter via `s.assigned_to`. Items-done count is the same as items but adds `i.ai_status = 'done'`. Exports count via `public.export_history eh join public.sessions s on s.id = eh.session_id` filtered by `eh.exported_at` between bounds, with same mode and specialist scoping.

  Return a single row via `select` of all 10 counts. Followed by `grant execute on function public.get_today_kpis(text[], text) to authenticated;`.

3. **Function 2 — `get_active_sessions(p_specialists text[] default array[]::text[], p_mode text default 'all')`** — right-now per D-15. Returns one row per active session with the specialist `display_name` joined server-side (Pitfall 1 — PostgREST cannot resolve cross-schema FK chain `sessions.assigned_to → auth.users → profiles`).

  Body shape:

  ```sql
  select
    s.id                            as session_id,
    s.name,
    s.mode,
    s.status,
    p.id                            as assigned_to_id,
    p.display_name                  as assigned_to_display_name,
  ```

7. **Function 6 — `get_house_sale_split(p_from, p_to, p_specialists, p_mode)`** — range-driven per D-17. Returns 2 rows (house, sale) via VALUES + LEFT JOIN so paired-KPI tile slots are stable:
  ```sql
  with modes(mode) as (values ('house'), ('sale')),
  -- scoped sessions
  scoped_sessions as (
    select s.mode, s.id
    from public.sessions s
    left join public.profiles p on p.id = s.assigned_to
    where s.created_at >= p_from and s.created_at < p_to
      and (cardinality(p_specialists) = 0 or p.email = any(p_specialists))
  ),
  scoped_items as (
    select s.mode, i.id
    from public.items i
    join public.sessions s on s.id = i.session_id
    left join public.profiles p on p.id = s.assigned_to
    where i.created_at >= p_from and i.created_at < p_to
      and (cardinality(p_specialists) = 0 or p.email = any(p_specialists))
  )
  select
    m.mode,
    coalesce(count(distinct ss.id), 0)::bigint as n_sessions,
    coalesce(count(si.id),         0)::bigint as n_items
  from modes m
  left join scoped_sessions ss on ss.mode = m.mode
  left join scoped_items    si on si.mode = m.mode
  group by m.mode
  order by m.mode;
  ```

  ```sql
  with scoped as (
    select i.id, i.category, s.mode, p.id as specialist_id, p.display_name as specialist_display_name
    from public.items i
    join public.sessions s on s.id = i.session_id
    left join public.profiles p on p.id = s.assigned_to
    where i.created_at >= p_from and i.created_at < p_to

      and (cardinality(p_specialists) = 0 or p.email = any(p_specialists))

  union all
  select 'mode' as dimension, mode as dim_key, mode as dim_label, count(*)::bigint

  select 'category' as dimension, coalesce(category, 'uncategorized') as dim_key,
  ```
