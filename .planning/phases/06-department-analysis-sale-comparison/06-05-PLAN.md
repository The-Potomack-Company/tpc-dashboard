    - Implementation: pass `payloadFilter={(row) => row.dataKey !== 'base'}` if ChartTooltip supports it; otherwise implement a lightweight inline tooltip component that reads `active` + `payload` + the matching `rows[i]` for the hovered step. Given UI-SPEC requires showing signed delta + running total, the CLEANEST path is a small inline tooltip component in this file — reusing `ChartTooltip` is not strictly required. Decision: inline a minimal tooltip function OR pass a custom `content` callback that reads the payload and renders the matching `WaterfallRow`.

    Chosen approach (for simplicity and test stability): implement an inline tooltip:
    ```tsx
    const WaterfallTooltip = ({ active, payload }: { active?: boolean; payload?: Array<{ payload: WaterfallRow }> }) => {
      if (!active || !payload || payload.length === 0) return null;
      const row = payload[0].payload;
      // For 'start' and 'end', suppress the running-total line (per UI-SPEC):
      const showRunningLine = row.direction !== 'start' && row.direction !== 'end';
      const sign = row.direction === 'down' ? '-' : row.direction === 'up' ? '+' : '';
      return (
        <div className="rounded-md bg-gray-900 text-white px-3 py-2 text-sm shadow-lg">
          <div className="font-semibold">{row.fullLabel}</div>
          <div className="tabular-nums">{sign}{formatCurrency(row.delta)}</div>
          {showRunningLine && (
            <div className="tabular-nums text-xs text-gray-300">Running total: {formatCurrency(row.runningTotal)}</div>
          )}
        </div>
      );
    };
    ```
    Wire via `<Tooltip content={<WaterfallTooltip />} />`.
